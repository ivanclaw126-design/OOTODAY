OOTODAY

先把衣橱导进来，今天穿什么才会开始变简单。

OOTODAY 是一个 AI wardrobe desk。它把衣橱导入、Today 推荐、买前分析和旅行打包接成一条更实用的日常决策链路。

天气 + 场景 + 已有衣橱，输出更像决策面板的穿搭建议。

Current Status

Import

低成本衣橱导入主线已经成型

支持单图、本地多图批量、商品链接 / 图片链接导入，以及拼图拆分后再进入同一条确认与保存链路。

Today

推荐页已经是可狗粮的核心体验

天气降级、城市修改、换一批推荐、OOTD 提交和最近历史都已经打通，并完成了一轮视觉精修。

Shop

买前分析已是可演示功能

可以分析商品链接或图片，判断重复风险、预计可搭套数和买不买建议，并已拦截非服饰类目。

Travel

旅行打包 MVP 已可保存与编辑

支持生成、保存、重新打开、删除和更新已保存方案，且页面层级已优化成更连续的出行决策流程。

Beta Checklist

朋友内测前的主线目标

01

让新用户先导入 3-5 件常穿衣服

把第一步压到最短，确保衣橱导入不会成为流失点。

02

让 Today 推荐尽快给出价值感

目标不是展示 AI，而是让用户觉得“这套我真的会穿”。

03

补上最小观测与反馈闭环

内测阶段最重要的是知道用户卡在哪、哪一步最有价值、出了问题能否快速定位。

Notes

关于 GitHub Pages 这页

这不是完整产品本体。当前 OOTODAY 是一个依赖 Next.js、Supabase 登录和服务端能力的动态应用， 所以 GitHub Pages 更适合作为公开进展页、分享页和朋友内测介绍页，而不是完整运行环境。

等你准备把真实应用公开给用户时，更适合部署到支持服务端运行的环境，比如 Vercel。
