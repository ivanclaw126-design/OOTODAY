初版规划：

下面我按“产品定位 → 用户分层 → 核心功能 → 关键流程 → AI能力 → 商业模式 → 路线图”帮你把这个 AI 穿搭应用补完整，尽量做到能直接拿去继续写 PRD。

⸻

一、先定义产品本质

你这个产品不只是“穿搭推荐工具”，更像一个：

AI 驱动的个人数字衣橱 + 穿搭决策助手 + 风格学习平台 + 购物参谋

它要解决的不是单点问题，而是一整条链路：

1. 我有哪些衣服
2. 这些衣服怎么搭
3. 别人怎么搭得好看
4. 我今天/明天/这个场景该穿什么
5. 缺什么该买什么，不该买什么
6. 买来的衣服怎么和现有衣橱结合
7. 我上身会是什么效果

所以产品目标应该是：

帮助用户以最低录入成本，建立可持续更新的数字衣橱，并通过 AI 实现“会整理、会学习、会推荐、会购买、会试穿”的闭环。

⸻

二、建议重新定义目标用户

不要一开始做“所有想穿搭的人”，而要切 3 类核心人群。

1. 效率型都市女性 / 男性

特点：

* 衣服不少，但每天出门还是不知道穿什么
* 有通勤、约会、旅行、健身等多场景
* 愿意为“省时间、减少买错”付费

核心需求：

* 快速搭配
* 天气/场景驱动推荐
* 避免重复买
* 出差/旅行打包建议

2. 内容学习型用户

特点：

* 喜欢看小红书、Pinterest、Instagram、抖音学穿搭
* 想模仿博主，但不知道自己衣橱里怎么落地
* 会保存很多图，但不会转化成实际穿搭

核心需求：

* 全网穿搭经验结构化学习
* 从 inspiration 到可执行搭配
* 根据自己已有单品“复刻”

3. 购买决策型用户

特点：

* 网购频繁
* 经常买了不会搭 / 色差严重 / 材质不符预期
* 希望买前先判断是否适合自己和现有衣橱

核心需求：

* 商品导入
* 和现有衣橱匹配度分析
* 购买建议、替代建议、性价比建议
* AI 上身预览

⸻

三、产品定位建议

你可以从下面三种定位里选一种为主，其他为辅。

方案 A：个人衣橱管家

强调：

* 衣橱管理
* 穿搭建议
* 购买建议
* OOTD 记录

适合：

* 最容易做 MVP
* 用户日常使用频率高

方案 B：AI 穿搭决策平台

强调：

* 从灵感学习到落地推荐
* 适合做社区/内容生态
* 更容易形成差异化

方案 C：AI 试穿 + 购物助手

强调：

* 商品导入
* 上身效果
* 购买判断
* 电商转化

适合：

* 后期商业化强
* 但前期技术和效果要求高

建议：前期用 A 做主线，B 做差异化，C 做商业化抓手。

⸻

四、核心产品价值主张

建议浓缩成 4 句话：

1. 最低成本导入你的真实衣橱
2. 把全网穿搭经验变成你能穿的方案
3. 每天给你适合天气、场景和心情的穿搭建议
4. 买前判断值不值得、买后马上融入你的衣橱

⸻

五、核心模块建议

⸻

模块 1：数字衣橱管理

这是产品地基，重点不是“管理”，而是低门槛导入。

1.1 衣橱导入方式

建议至少做 5 种：

A. 拍照导入

用户直接拍自己的衣服
AI 自动识别：

* 品类：衬衫 / T恤 / 西装 / 半裙 / 鞋 / 包
* 颜色
* 图案
* 材质感知
* 季节属性
* 风格标签
* 版型标签

B. 相册批量导入

用户从手机相册选多张
适合导入以前拍过的穿搭、单品图

C. 电商链接导入

支持淘宝 / 天猫 / 京东 / 小红书商品卡 / 抖音商城 / Amazon / SHEIN 等
抓取：

* 商品主图
* SKU 颜色
* 品类
* 材质
* 品牌
* 价格
* 尺码信息
* 商品详情图

D. 订单截图 / 订单同步导入

用户授权读取电商订单邮件/截图/订单页
自动识别买过的衣服
这是非常高价值的“懒人导入”方式

E. 拼图导入

这是你提到的重点，应该单独做成特色能力：

* 自动识别一张拼图里的多个商品
* 拆出每件单品
* 自动去背景
* 自动纠正压缩导致的色偏
* 合并同款不同角度图为一个商品档案

1.2 单品信息结构

每件衣服建议记录：

基础信息：

* 品类
* 子品类
* 品牌
* 颜色
* 面料
* 季节
* 场景
* 价格
* 购入时间
* 新旧程度
* 穿着频率

风格信息：

* 极简 / 通勤 / 韩系 / 法式 / 美式休闲 / 运动 / 甜酷 等

版型信息：

* 修身 / 宽松 / 高腰 / 落肩 / 直筒 / A字 等

搭配属性：

* 可搭配单品类型
* 难搭程度
* 百搭指数
* 显瘦/显高标签
* 正式度

1.3 衣橱管理能力

* 分类浏览
* 搜索筛选
* 胶囊衣橱整理
* 闲置识别
* 重复购买提醒
* 换季整理建议
* 缺失单品提示

⸻

模块 2：全网穿搭经验学习

这是你区别于普通衣橱 App 的关键。

2.1 内容来源

* 小红书
* 抖音
* Instagram
* Pinterest
* YouTube Shorts
* 博客 / 杂志 / 品牌 lookbook

2.2 学习的重点不是“看内容”，而是“结构化理解”

AI 要做三层处理：

第一层：内容理解

识别每条穿搭内容的：

* 风格
* 场景
* 季节
* 色彩组合
* 单品组合
* 搭配逻辑
* 适合体型
* 适合肤色
* 适合气质

第二层：经验提炼

把博主经验转成规则：

* 小个子怎么显高
* 苹果型身材如何穿
* 通勤如何兼顾正式和轻松
* 基础款如何穿出质感
* 某颜色怎么避免土气

第三层：个性化映射

结合用户衣橱说：

* 你可以直接复刻哪套
* 你缺哪件关键单品
* 用你现有衣服最接近的替代方案是什么

2.3 内容产品形态

建议不要做成泛社区，先做“AI 学习流”：

* 今日灵感
* 为你拆解的穿搭逻辑
* 从博主图到你的衣橱复刻
* 一键保存为“想试试”
* 一键生成“我的版本”

⸻

模块 3：AI 穿搭建议

这是用户最高频使用场景。

3.1 推荐维度

系统每天推荐时要结合：

* 当地天气
* 温度
* 降雨
* 风力
* 日程场景
* 用户风格偏好
* 身材特征
* 今天心情/状态
* 穿过频率
* 洗衣状态
* 近期照片避免重复

3.2 推荐场景

* 今日通勤穿搭
* 约会穿搭
* 旅行穿搭
* 面试穿搭
* 会议穿搭
* 周末逛街
* 雨天穿搭
* 换季穿搭
* 突发场景：“今晚临时聚餐穿什么”

3.3 推荐输出形式

每次推荐不要只给一套，建议给 3 层答案：

A. 最稳妥

适合大多数时候，不容易出错

B. 更出彩

更时髦、更有记忆点

C. 最省力

用户今天赶时间时的一键方案

每套方案展示：

* 单品组合
* 搭配理由
* 适合场景
* 保暖/舒适度
* 可替换单品
* 缺失单品建议

3.4 交互方式

用户输入可以很自然：

* 今天天气 18 度有点风，去见客户，想显得专业但别太严肃
* 晚上约会，想温柔一点
* 我今天不想穿裙子
* 想穿这双鞋，帮我配

这类对话式输入会非常重要。

⸻

模块 4：购买建议

这是未来商业化核心。

4.1 购买前判断

导入商品后，AI 给出：

* 和现有衣橱匹配度
* 可组成多少套搭配
* 是否与你已有单品重复
* 实用频率预测
* 是否值得买
* 价格是否合理
* 是否更推荐替代款

4.2 购买建议输出模板

建议标准化成：

这件适不适合你？

* 风格匹配度：高 / 中 / 低
* 衣橱兼容度：可搭 8 件现有单品
* 使用场景：通勤 / 周末 / 旅行
* 风险点：颜色挑肤色 / 版型挑身材 / 材质易皱
* 建议：可以买 / 可等折扣 / 不建议买 / 更推荐替代款

4.3 替代建议

* 同风格更高性价比
* 同预算更好搭
* 更适合用户体型
* 更适合已有衣橱缺口

4.4 购物车助手

可做成很强的场景：

* 批量分析购物车
* 告诉用户哪几件更值得留
* 哪几件功能重叠
* 哪件是冲动消费

⸻

模块 5：OOTD 记录与反馈闭环

很多穿搭类产品容易忽略这一块，但它是模型长期变准的关键。

5.1 OOTD 录入方式

* 自拍上传
* 全身镜照片上传
* 从相册选
* 从社交平台同步
* 一键从今日推荐生成“已穿”

5.2 OOTD 信息结构

* 日期
* 场景
* 天气
* 穿着单品
* 是否满意
* 是否收到夸奖
* 舒适度
* 上镜效果
* 再穿意愿

5.3 价值

OOTD 不是日记，而是训练数据：

* 用户真实偏好
* 穿搭接受度
* 单品使用率
* 推荐效果反馈
* “好看但不舒服” vs “舒服但不够出彩”

5.4 AI 复盘

每周/每月给用户：

* 最常穿的单品
* 最成功搭配
* 最闲置单品
* 风格变化趋势
* 购买浪费分析
* 下阶段建议补货单品

⸻

模块 6：AI 试穿 / 上身效果预览

这是很有吸引力但技术要求高的模块，建议分阶段做。

6.1 能力分层

第一阶段：静态搭配预览

* 用单品图组合成平铺穿搭图
* 先不追求真人上身

第二阶段：用户形象虚拟试穿

* 用户上传身材照片 / 基础体型信息
* 展示服装上身大致效果

第三阶段：高保真试穿

* 体型
* 肤色
* 面料垂坠感
* 光照
* 动态效果

6.2 现实建议

初期不要承诺“绝对真实”，而要定义为：
风格预览 / 搭配参考 / 版型趋势参考

因为真正的：

* 色差
* 材质
* 垂坠
* 贴合度
* 真实体感
    都很难在早期做到足够可信。

6.3 最适合的应用点

* 买前预览
* 复刻博主穿搭
* 看不同颜色哪个更适合
* 看这件衣服和现有衣橱是否协调

⸻

六、你最该重视的“低成本输入”设计

因为这类产品最容易死在：用户懒得录入。

所以产品成败很大程度取决于：
能不能让用户 3 天内完成 70% 衣橱初始化，而不觉得累。

建议做的导入策略

1. 新用户 onboarding 只做最小集

先别要求录完整衣橱，只让用户先导入：

* 10 件上衣
* 5 件下装
* 3 双鞋
* 2 个包
* 2 套常穿搭配

这样先能出推荐。

2. 从“最近常穿”开始

比“全量衣橱录入”更现实。

3. 用任务机制引导补录

例如：

* 再录入 5 件单品，解锁更准推荐
* 录入 3 条 OOTD，生成你的个人风格画像
* 同步购物记录，获取避免重复购买建议

4. 自动补全

用户拍一张图后，AI 自动：

* 去背景
* 分品类
* 估颜色
* 估季节
* 补标签
    用户只要确认即可

⸻

七、建议把产品做成“两个主界面”

1. Today 页

高频使用页，解决今天穿什么
包括：

* 今日天气
* 今日推荐 3 套
* 快速输入需求
* 待洗 / 不想穿 / 指定某件单品
* 今日 OOTD 记录入口

2. Closet 页

管理和资产沉淀页
包括：

* 我的单品
* 搭配收藏
* 待购清单
* 闲置预警
* 风格分析
* OOTD 历史

3. Inspiration 页

学习页
包括：

* 为你推荐的全网穿搭
* AI 拆解逻辑
* 可复刻程度
* 一键生成“我的版本”

4. Shop 页

购买助手页
包括：

* 商品导入
* 购买分析
* 替代推荐
* 购物车诊断

八、AI 能力拆解

这个产品背后的 AI 不止一个模型，而是一组能力。

1. 视觉识别

识别衣物：

* 品类
* 颜色
* 图案
* 材质倾向
* 轮廓
* 风格

2. 图像处理

* 去背景
* 拼图拆分
* 色彩校正
* 商品主图高清化
* 同款多图聚合

3. 推荐系统

结合：

* 用户画像
* 现有衣橱
* 历史 OOTD
* 场景/天气
* 购买行为

4. 多模态理解

看图 + 看文案 + 看商品详情
理解一套穿搭的逻辑

5. 生成能力

* 穿搭文案解释
* 购买建议解释
* 搭配图生成
* 虚拟试穿图生成

6. 长期记忆

记住：

* 用户喜欢什么
* 不喜欢什么
* 什么推荐被采纳了
* 哪类商品买后后悔

九、建议重点定义几个“超级场景”

不要平均发力，先抓高频高价值场景。

超级场景 1：明天穿什么

输入：

* 天气
* 日程
* 心情
    输出：
* 3 套穿搭
* 替代方案
* 缺失补齐建议

超级场景 2：这件值不值得买

输入：

* 商品链接 / 拼图 / 截图
    输出：
* 值不值得买
* 能搭几套
* 是否重复
* 替代建议

超级场景 3：复刻这张博主图

输入：

* 一张 inspiration 图
    输出：
* 搭配拆解
* 用现有衣橱复刻
* 缺少哪些单品
* 生成我的版本

超级场景 4：整理衣橱

输出：

* 闲置清单
* 重复单品
* 缺失核心基础款
* 胶囊衣橱方案

超级场景 5：旅行打包

输入：

* 城市
* 天数
* 行程类型
    输出：
* 带哪些单品
* 组合成多少套
* 少带但不重样

十、MVP 建议怎么收

不要一开始做太重。

MVP 版本建议只做这 4 个核心能力

1. 低成本衣橱导入

* 拍照
* 相册
* 电商链接
* 拼图拆分基础版

2. 今日穿搭建议

* 天气 + 场景 + 衣橱推荐

3. 商品购买分析

* 导入商品
* 输出兼容度 + 搭配数 + 是否建议买

4. OOTD 记录

* 照片上传
* 满意度反馈
* 周报

这四个就足够形成闭环。

十二、商业模式建议

1. 订阅制

适合做主收入。
免费版：

* 基础衣橱管理
* 少量推荐

会员版：

* 高级穿搭建议
* 购买分析
* 虚拟试穿
* 风格周报/月报
* 跨平台商品导入

2. 联盟电商佣金

当用户购买推荐商品时抽佣。

3. 品牌合作

* 品牌试穿
* 品牌定制搭配专题
* 新品匹配推荐

4. 高级数字服务

* 一对一 AI 形象顾问
* AI 胶囊衣橱规划
* 旅行打包包

十三、关键指标建议

北极星指标

用户每周被采纳的穿搭/购买决策次数

因为真正价值不在浏览，而在“帮用户做出决定”。

核心指标

* 新用户 7 日内衣橱导入件数
* 每周活跃用户的推荐查看率
* 推荐采纳率
* 商品分析使用率
* OOTD 记录率
* 30 日留存

十四、最容易踩的坑

1. 录入太麻烦

会直接导致流失。

2. 推荐太“像模板”

用户会觉得不懂我。

3. AI 试穿不真实

会迅速损害信任。

4. 学习内容太泛

如果只是“给我看博主图”，没有价值。

5. 商品分析不够具体

“适合你”这类空话没有说服力，必须讲清为什么。
